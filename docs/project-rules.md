# Project Rules

## Goal

Maximize the evaluation score for the Ascendra Workspaces take-home assignment within the expected 4–6 hour implementation scope.

---

## General Principles

- Follow the take-home assignment first.
- Follow the Design System as the visual source of truth.
- Prefer simple, production-quality solutions.
- Do not over-engineer.
- Keep the project clean and readable.

---

## Technology

- React
- TypeScript
- Vite
- Tailwind CSS
- shadcn/ui

---

## Code Quality

- Reusable components.
- Strong typing.
- Clear folder structure.
- No unnecessary abstractions.
- Consistent naming.
- Small focused components.

---

## UI / UX

- Responsive (desktop first).
- Enterprise dashboard aesthetics.
- High information density.
- Accessibility.
- Consistent spacing and typography.
- Follow the provided color palette and design tokens.

---

## Scope

Implement only what is required by the assignment.

Do not add extra features unless they clearly improve the evaluation score.

---

## Workflow

Read all project documentation before implementation.

Implement one milestone at a time.

After each milestone:

- explain decisions;
- wait for approval before continuing.
