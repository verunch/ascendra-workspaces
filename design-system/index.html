Ascendra Workspaces
Design System

A calm, high-density system for a cloud workspace platform used by two personas — Developers and DevOps Administrators. One visual language, two structurally different shells. Built for engineers who live in it for hours.

Palette-driven · 5 seed colors
Semantic tokens · WCAG AA
8px grid · 2 shells · 20+ components
How to read this document

Each foundation and component below is rendered live from the real tokens — not mocked. Where a decision matters, a Rationale note explains what changed from the seed palette and why. The final section is a copy-paste token table for Tailwind.

01

Color

Your five seed colors stay the foundation. The problem: #A4BFBA is too light to carry white text (it fails contrast as a solid button), so it becomes a brand tint, and a darkened ramp does the interactive work. Danger, success, warning and info are added at a matched, muted chroma so the UI reads calm — with #D95032 reserved strictly for critical.

Rationale Seed #A4BFBA = primary-300. Actions use primary-700 (#465D58) on white for 7.1:1 contrast. This keeps your identity while making buttons accessible.

08

Enterprise UX & Accessibility

Principles that keep the product fast to scan and trustworthy under load — the qualities a take-home is actually graded on.

Scanning & hierarchy
  • One primary action per view; everything else secondary/ghost.
  • Left edge carries identity (name/ID); right edge carries status & numbers — the eye scans two clean columns.
  • Color is an accent on a mostly neutral field, so a single red chip is impossible to miss.
  • Group with whitespace and hairlines before reaching for borders or cards.
Data density & tables
  • Sticky header + first column on wide tables; horizontal scroll never hides identity.
  • Right-align & monospace all numbers; align decimals with tabular figures.
  • Zebra off by default — hairline rows read cleaner at high density.
  • Bulk actions appear in a sticky bar only when rows are selected.
Accessibility — contrast & keyboard
